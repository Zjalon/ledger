# 统计页面重新设计

## 目标

重新设计统计页面（Stat.vue），用更合理的信息架构替代现有四个板块，消除冗余，统一时间控件。

## 变更概述

| 操作 | 内容 |
|------|------|
| 移除 | "收支构成" 饼图（两扇区，信息量低） |
| 移除 | "分类与成员" 饼图（与"本月成员构成"重叠） |
| 移除 | "本月成员构成" 饼图（冗余） |
| 移除 | "本月每日趋势" 硬编码当月的折线图 |
| 新增 | 全局时间选择器（本周/本月/今年） |
| 新增 | 概览卡片（支出/收入/结余） |
| 新增 | 趋势折线图（联动全局时间，支持支出/收入切换） |
| 新增 | 分类排行列表（带占比条，支持成员筛选） |

## 页面布局

从上到下：

1. **Ledger 外壳顶栏** — 品牌 + 同步状态（不变）
2. **全局时间选择器** — 三个 tabs：本周 / 本月 / 今年
3. **概览卡片** — 支出 / 收入 / 结余，横向三张
4. **趋势折线图** — SVG 手绘，支持支出/收入切换
5. **分类排行** — 列表 + 占比条，支持成员筛选

## 全局时间选择器

页面顶部，三个 tabs 横向排列：

- **本周**：周一 00:00 → 今天 23:59
- **本月**：1号 00:00 → 今天 23:59
- **今年**：1月1日 00:00 → 今天 23:59

选中后，下方所有板块联动更新。不做自定义日期范围。

## 概览卡片

替代原来的"收支构成"饼图，三张小卡片横向排列：

- **支出**：红色数字，带负号前缀
- **收入**：绿色数字，带正号前缀
- **结余**：正数绿色、负数红色、零灰色
- 金额通过 `amountToNumber` 转换（已有 10000:1 比例）
- 不显示笔数，纯数字展示
- 无点击交互

## 趋势折线图

替代原来的"本月每日趋势"，联动全局时间选择器：

| 全局时间 | X 轴粒度 | 数据点数 |
|----------|---------|---------|
| 本周 | 每天（周一→周日） | 7 |
| 本月 | 每天（1号→今天） | 最多 31 |
| 今年 | 每月（1月→当前月） | 最多 12 |

- SVG 手绘折线图，下方带面积填充（半透明渐变）
- 顶部小 tabs：**支出** / **收入**，切换显示不同类型
- X 轴显示日期标签（本周：周一/周二…，本月：1/2/3…，今年：1月/2月…）
- 数据点 hover/点击显示具体金额
- 继续使用现有 SVG 方案，不引入第三方图表库

## 分类排行

替代原来的"分类与成员"和"本月成员构成"两个板块：

- 顶部小 tabs：**支出** / **收入**
- 成员筛选：一行 chips，点击切换（全部 / 单个成员）
- 主体：分类列表，按金额从高到低排序
- 每行显示：分类名称、金额（右对齐）、占比条（横向半透明条，长度按百分比）
- 不使用饼图，分类多时饼图扇区太碎，列表更清晰

## 数据依赖

- 交易数据：通过 `ep.tableGetAllItems(bookId, "transactions")` 获取，已有
- 成员列表：通过 `ep.tableGetAllItems(bookId, "users")` 获取，已有
- 分类数据：通过 `ep.getLedgerMeta(bookId)` 获取，已有
- 所有数据已在 `Stat.vue` 中加载，无需新增数据源
